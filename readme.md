# 테니스 게임

Java로 구현하는 콘솔 기반 테니스 경기 점수 계산 프로그램입니다. 사용자는 경기 방식과 선수 이름을 입력하고, 각 포인트를 획득한 선수를 반복 입력합니다. 프로그램은 테니스 규칙에 따라 포인트, 게임, 세트, 매치 스코어를 계산하고 현재 스코어보드를 콘솔에 출력합니다.

## 개발 환경

- 언어: Java
- 실행 화면: 콘솔
- 입력 방식: 키보드 입력
- 출력 방식: 콘솔 스코어보드 출력

## 경기 설정

프로그램 시작 시 다음 정보를 입력받습니다.

- 경기 세트 수
  - 3세트 경기: 2세트를 먼저 이긴 선수가 최종 승리
  - 5세트 경기: 3세트를 먼저 이긴 선수가 최종 승리
- 선수 1 이름
- 선수 2 이름

## 주요 클래스 요구사항

테니스 경기의 점수를 계산하는 계수기 클래스를 선언합니다.

### 저장해야 할 정보

- 경기 세트 수
- 선수 이름
- 각 선수가 승리한 세트 수
- 각 세트에서 각 선수가 승리한 게임 수
- 현재 게임의 포인트
- 현재 경기가 종료되었는지 여부
- 최종 승자

### 필수 메서드

#### `pointWinner(int p)`

포인트를 획득한 선수를 입력받아 점수를 갱신합니다.

- `p`가 `1`이면 선수 1의 포인트 증가
- `p`가 `2`이면 선수 2의 포인트 증가
- 현재 게임 포인트를 계산
- 게임 승리 조건을 만족하면 게임 스코어 갱신
- 세트 승리 조건을 만족하면 세트 스코어 갱신
- 매치 승리 조건을 만족하면 최종 승자 결정

#### `dispScoreBoard()`

현재 경기 스코어보드를 콘솔에 출력합니다.

출력 내용에는 다음 정보가 포함되어야 합니다.

- 선수 이름
- 세트 스코어
- 현재 세트의 게임 스코어
- 현재 게임의 포인트
- 듀스, 어드밴티지, 타이브레이크 상태
- 경기 종료 시 최종 승자

## 점수 규칙

### 포인트 표기

일반 게임의 포인트는 다음 순서로 표시합니다.

| 내부 포인트 수 | 테니스 점수 |
| --- | --- |
| 0 | 0 |
| 1 | 15 |
| 2 | 30 |
| 3 | 40 |

예시:

- 선수 1이 2포인트, 선수 2가 1포인트이면 `30 - 15`
- 선수 1이 3포인트, 선수 2가 0포인트이면 `40 - 0`

### 일반 게임 승리

한 선수가 4포인트 이상을 얻고, 상대보다 2포인트 이상 앞서면 해당 게임을 승리합니다.

예시:

- `40 - 0` 상태에서 선수 1이 포인트를 얻으면 선수 1 게임 승리
- `40 - 30` 상태에서 선수 1이 포인트를 얻으면 선수 1 게임 승리

### 듀스

두 선수가 모두 3포인트 이상이고 점수가 같으면 듀스입니다.

예시:

- `40 - 40`
- 양쪽 모두 4포인트 이상이며 같은 점수인 상태

듀스 이후에는 한 선수가 연속으로 상대보다 2포인트 앞서야 게임을 승리합니다.

### 어드밴티지

듀스 이후 한 선수가 상대보다 1포인트 앞서면 어드밴티지 상태입니다.

예시:

- 선수 1이 앞서면 `Advantage 선수1`
- 선수 2가 앞서면 `Advantage 선수2`

어드밴티지 상태에서 앞선 선수가 다음 포인트를 얻으면 게임을 승리합니다. 상대 선수가 포인트를 얻으면 다시 듀스가 됩니다.

## 세트 규칙

한 선수가 6게임 이상을 얻고, 상대보다 2게임 이상 앞서면 해당 세트를 승리합니다.

예시:

- `6 - 0`: 세트 승리
- `6 - 4`: 세트 승리
- `7 - 5`: 세트 승리

단, 게임 스코어가 `6 - 6`이 되면 타이브레이크를 진행합니다.

## 타이브레이크 규칙

타이브레이크는 세트 게임 스코어가 `6 - 6`일 때 적용합니다.

- 타이브레이크 점수는 일반 테니스 포인트가 아니라 `0, 1, 2, 3...` 형태의 숫자로 표시합니다.
- 한 선수가 7점 이상을 얻고, 상대보다 2점 이상 앞서면 타이브레이크를 승리합니다.
- 타이브레이크 승자는 해당 세트를 `7 - 6`으로 승리합니다.

예시:

- 타이브레이크 `7 - 3`: 세트 승리
- 타이브레이크 `8 - 6`: 세트 승리
- 타이브레이크 `10 - 8`: 세트 승리

## 매치 승리 규칙

경기 세트 수에 따라 필요한 세트 승리 수가 달라집니다.

| 경기 방식 | 최종 승리 조건 |
| --- | --- |
| 3세트 경기 | 2세트 선승 |
| 5세트 경기 | 3세트 선승 |

필요한 세트 수를 먼저 승리한 선수가 최종 승자가 되며, 프로그램은 최종 스코어보드와 승자 이름을 출력하고 종료합니다.

## 프로그램 동작 흐름

1. 콘솔에서 경기 세트 수를 입력받습니다.
2. 선수 1과 선수 2의 이름을 입력받습니다.
3. 테니스 경기 계수기 객체를 생성합니다.
4. 포인트를 획득한 선수를 입력받습니다.
5. `pointWinner(int p)`를 호출해 점수를 갱신합니다.
6. `dispScoreBoard()`를 호출해 현재 스코어보드를 출력합니다.
7. 최종 승자가 결정될 때까지 4~6번 과정을 반복합니다.
8. 경기가 종료되면 최종 스코어보드와 승자 이름을 출력합니다.

## 입력 예시

```text
경기 세트 수를 입력하세요(3 또는 5): 3
선수 1 이름: Nadal
선수 2 이름: Federer

포인트를 획득한 선수 번호를 입력하세요(1 또는 2): 1
```

## 출력 예시

```text
================================
현재 스코어보드
경기 방식: 3세트

선수        세트   현재 세트 게임   현재 게임 포인트
Nadal       0      0               15
Federer     0      0               0
================================
```

듀스 상태 예시:

```text
현재 게임 포인트: Deuce
```

어드밴티지 상태 예시:

```text
현재 게임 포인트: Advantage Nadal
```

타이브레이크 상태 예시:

```text
현재 세트: Tie-break
타이브레이크 점수: Nadal 7 - Federer 6
```

경기 종료 예시:

```text
경기 종료
최종 승자: Nadal
최종 세트 스코어: Nadal 2 - Federer 1
```

## 구현 시 유의사항

- 잘못된 세트 수가 입력되면 다시 입력받습니다.
- 포인트 획득 선수 번호가 `1` 또는 `2`가 아니면 다시 입력받습니다.
- 게임이 끝나면 현재 게임 포인트를 초기화합니다.
- 세트가 끝나면 현재 세트의 게임 스코어를 초기화하고 다음 세트를 시작합니다.
- 타이브레이크가 끝나면 해당 세트를 종료합니다.
- 최종 승자가 결정된 뒤에는 추가 입력을 받지 않습니다.
