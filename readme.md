# 테니스 게임

Java로 구현하는 콘솔 기반 테니스 경기 점수 시뮬레이션 프로그램입니다. 사용자는 경기 방식, 단식/복식, 참가 선수를 선택하면 프로그램이 테니스 규칙에 따라 매치를 자동으로 랜덤 시뮬레이션합니다. 경기 결과는 콘솔에 최종 스코어보드와 승자로 출력되며 파일로 저장됩니다. 선수 이름으로 과거 게임 기록을 조회할 수 있습니다.

## 개발 환경

- 언어: Java
- 실행 화면: 콘솔
- 입력 방식: 키보드 입력
- 출력 방식: 콘솔 스코어보드 출력 + 파일 저장

---

## 기능 목록

| 기능 | 설명 |
| --- | --- |
| 게임 입력 | 세트 수, 단식/복식, 참가 선수 선택 |
| 게임 플레이 | 랜덤 시뮬레이션 → 최종 스코어보드 및 승자 출력, 파일 저장 |
| 게임 조회 | 선수 이름 입력 → 해당 선수의 게임 기록 조회 |

---

## 게임 입력

프로그램 시작 시 메뉴에서 기능을 선택하고, 게임 플레이를 선택하면 다음 정보를 순서대로 입력합니다.

### 1. 경기 세트 수

| 입력값 | 경기 방식 | 최종 승리 조건 |
| --- | --- | --- |
| `3` | 3세트 경기 | 2세트 선승 |
| `5` | 5세트 경기 | 3세트 선승 |

잘못된 값이 입력되면 다시 입력받습니다.

### 2. 단식 / 복식 선택

| 입력값 | 경기 유형 | 선수 선택 수 |
| --- | --- | --- |
| `1` | 단식 (Singles) | 2명 선택 |
| `2` | 복식 (Doubles) | 4명 선택 (팀당 2명) |

### 3. 선수 선택

등록된 선수 목록에서 번호로 선택합니다.

| 번호 | 선수 이름 |
| --- | --- |
| 1 | Novak Djokovic |
| 2 | Rafael Nadal |
| 3 | Roger Federer |
| 4 | Carlos Alcaraz |
| 5 | Jannik Sinner |
| 6 | Daniil Medvedev |
| 7 | Alexander Zverev |
| 8 | Andrey Rublev |
| 9 | Casper Ruud |
| 10 | Stefanos Tsitsipas |

- **단식**: 선수 1명, 선수 2명 순서로 선택 (같은 선수 중복 선택 불가)
- **복식**: 팀 A 선수 2명, 팀 B 선수 2명 순서로 선택 (중복 선택 불가)

---

## 게임 플레이

선수 선택이 완료되면 경기가 자동으로 진행됩니다. 사용자가 별도로 포인트를 입력하지 않으며, 매 포인트는 랜덤하게 결정됩니다.

- 매 포인트마다 양 팀 중 한 쪽이 랜덤으로 득점합니다.
- 테니스 규칙(포인트 → 게임 → 세트 → 매치)에 따라 자동으로 스코어가 계산됩니다.
- 최종 승자가 결정되면 최종 스코어보드와 승자를 콘솔에 출력합니다.
- 경기 결과는 파일로 저장됩니다.

---

## 게임 조회

선수 이름을 입력하면 해당 선수가 참가한 게임 기록을 조회합니다.

- 저장된 파일에서 해당 선수의 기록을 읽어 출력합니다.
- 게임 날짜, 경기 방식, 상대 선수(팀), 세트 스코어, 승/패 여부를 표시합니다.
- 기록이 없을 경우 안내 메시지를 출력합니다.

---

## 주요 클래스 요구사항

테니스 경기의 점수를 계산하는 계수기 클래스를 선언합니다.

### 저장해야 할 정보

- 경기 세트 수
- 단식/복식 여부
- 선수(팀) 이름
- 각 선수(팀)가 승리한 세트 수
- 각 세트에서 각 선수(팀)가 승리한 게임 수
- 현재 게임의 포인트
- 현재 경기가 종료되었는지 여부
- 최종 승자

### 필수 메서드

#### `pointWinner(int p)`

포인트를 획득한 팀을 받아 점수를 갱신합니다.

- `p`가 `1`이면 팀 1의 포인트 증가
- `p`가 `2`이면 팀 2의 포인트 증가
- 현재 게임 포인트를 계산
- 게임 승리 조건을 만족하면 게임 스코어 갱신
- 세트 승리 조건을 만족하면 세트 스코어 갱신
- 매치 승리 조건을 만족하면 최종 승자 결정

> 랜덤 시뮬레이션 시 `Math.random()`으로 1 또는 2를 결정하여 반복 호출합니다.

#### `dispScoreBoard()`

현재 경기 스코어보드를 콘솔에 출력합니다.

출력 내용에는 다음 정보가 포함되어야 합니다.

- 선수(팀) 이름
- 세트 스코어
- 현재 세트의 게임 스코어
- 현재 게임의 포인트
- 듀스, 어드밴티지, 타이브레이크 상태
- 경기 종료 시 최종 승자

## 점수 규칙

### 포인트 표기

일반 게임의 포인트는 다음 순서로 표시합니다.

| 내부 포인트 수 | 테니스 점수 |
| --- | --- |
| 0 | 0 |
| 1 | 15 |
| 2 | 30 |
| 3 | 40 |

예시:

- 팀 1이 2포인트, 팀 2가 1포인트이면 `30 - 15`
- 팀 1이 3포인트, 팀 2가 0포인트이면 `40 - 0`

### 일반 게임 승리

한 팀이 4포인트 이상을 얻고, 상대보다 2포인트 이상 앞서면 해당 게임을 승리합니다.

예시:

- `40 - 0` 상태에서 팀 1이 포인트를 얻으면 팀 1 게임 승리
- `40 - 30` 상태에서 팀 1이 포인트를 얻으면 팀 1 게임 승리

### 듀스

두 팀이 모두 3포인트 이상이고 점수가 같으면 듀스입니다.

예시:

- `40 - 40`
- 양쪽 모두 4포인트 이상이며 같은 점수인 상태

듀스 이후에는 한 팀이 연속으로 상대보다 2포인트 앞서야 게임을 승리합니다.

### 어드밴티지

듀스 이후 한 팀이 상대보다 1포인트 앞서면 어드밴티지 상태입니다.

예시:

- 팀 1이 앞서면 `Advantage 팀1`
- 팀 2가 앞서면 `Advantage 팀2`

어드밴티지 상태에서 앞선 팀이 다음 포인트를 얻으면 게임을 승리합니다. 상대 팀이 포인트를 얻으면 다시 듀스가 됩니다.

## 세트 규칙

한 팀이 6게임 이상을 얻고, 상대보다 2게임 이상 앞서면 해당 세트를 승리합니다.

예시:

- `6 - 0`: 세트 승리
- `6 - 4`: 세트 승리
- `7 - 5`: 세트 승리

단, 게임 스코어가 `6 - 6`이 되면 타이브레이크를 진행합니다.

## 타이브레이크 규칙

타이브레이크는 세트 게임 스코어가 `6 - 6`일 때 적용합니다.

- 타이브레이크 점수는 일반 테니스 포인트가 아니라 `0, 1, 2, 3...` 형태의 숫자로 표시합니다.
- 한 팀이 7점 이상을 얻고, 상대보다 2점 이상 앞서면 타이브레이크를 승리합니다.
- 타이브레이크 승자는 해당 세트를 `7 - 6`으로 승리합니다.

예시:

- 타이브레이크 `7 - 3`: 세트 승리
- 타이브레이크 `8 - 6`: 세트 승리
- 타이브레이크 `10 - 8`: 세트 승리

## 매치 승리 규칙

경기 세트 수에 따라 필요한 세트 승리 수가 달라집니다.

| 경기 방식 | 최종 승리 조건 |
| --- | --- |
| 3세트 경기 | 2세트 선승 |
| 5세트 경기 | 3세트 선승 |

필요한 세트 수를 먼저 승리한 팀이 최종 승자가 되며, 프로그램은 최종 스코어보드와 승자 이름을 출력하고 파일로 저장합니다.

## 프로그램 동작 흐름

### 메인 메뉴

```
1. 게임 플레이
2. 게임 기록 조회
3. 종료
```

### 게임 플레이 흐름

1. 경기 세트 수를 입력받습니다 (3 또는 5).
2. 단식 또는 복식을 선택합니다.
3. 선수 목록을 출력하고 참가 선수(팀)를 번호로 선택합니다.
4. 테니스 경기 계수기 객체를 생성합니다.
5. 랜덤으로 득점 팀을 결정하여 `pointWinner(int p)`를 호출합니다.
6. 최종 승자가 결정될 때까지 5번 과정을 반복합니다.
7. `dispScoreBoard()`를 호출해 최종 스코어보드와 승자를 출력합니다.
8. 경기 결과를 파일로 저장합니다.

### 게임 조회 흐름

1. 조회할 선수 이름을 입력받습니다.
2. 저장 파일에서 해당 선수가 참가한 경기 기록을 검색합니다.
3. 검색된 기록을 날짜순으로 출력합니다.

## 파일 저장 형식

경기 결과는 `results.txt` 파일에 누적 저장됩니다.

```text
DATE=2026-06-10
TYPE=단식
SETS=3
PLAYER1=Nadal
PLAYER2=Federer
SCORE=6-3,6-4
WINNER=Nadal
---
```

## 구현 시 유의사항

- 잘못된 세트 수가 입력되면 다시 입력받습니다.
- 잘못된 경기 유형(단식/복식)이 입력되면 다시 입력받습니다.
- 선수 선택 시 유효하지 않은 번호나 중복 선택이 발생하면 다시 입력받습니다.
- 복식 선택 시 팀 A와 팀 B를 구분하여 각 팀 2명씩 선택합니다.
- 게임이 끝나면 현재 게임 포인트를 초기화합니다.
- 세트가 끝나면 현재 세트의 게임 스코어를 초기화하고 다음 세트를 시작합니다.
- 타이브레이크가 끝나면 해당 세트를 종료합니다.
- 경기 결과는 `results.txt`에 누적 저장되며, 조회 시 해당 파일을 읽어 선수 이름으로 필터링합니다.


## 추가 설명

### 게임점수 획득시 처리

```
simulate() → pointWinner(p)
                └─ currentSet.pointWinner(p)
                        │
                   currentSet.isOver() == true (세트 종료)
                        │
                   setsWon[winner]++
                        │
                   setsWon >= setsToWin ?
                        ├─ YES → matchOver = true  ← 여기서 발생
                        └─ NO  → currentSet = new SetScore() (다음 세트 시작)
```

듀스는 `GameScore.checkWin()`에서 처리하고, `getPointDisplay()`에서 듀스에 대한 출력을 표시함.

### 타이브레이크

```
게임 스코어가 6-6 도달
    │
    └─ SetScore.checkTiebreak()
            └─ currentGame = new GameScore(true)  ← isTiebreak=true로 생성
                    │
                    포인트 추가 반복
                    │
              GameScore.checkWin()
              7점 이상 AND 2점 차 → 게임 종료
                    │
              SetScore: games → 7-6
              checkSetWin() → setOver = true
```
